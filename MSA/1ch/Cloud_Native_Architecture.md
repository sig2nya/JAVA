1. 확장 가능한 아키텍처
- 시스템 수평적 확장에 유연
- 확장된 서버로 시스템의 부하 분산, 가용성 보장
- 시스템 혹은 서비스 애플리케이션 단위의 패키지 (컨테이너 기반 패키지)
- 모니터링

2. 탄력적 아키텍처
- 서비스 생성 - 통합 - 배포, 비즈니스 환경 변화에 신속한 대응
- 분할된 서비스 구조
- 무상태 통신(Stateless) Protocol
- 서비스의 추가 및 삭제 자동 감지
- 변경된 서비스 요청에 따라 사용자 요청 처리 (동적 처리)

3. 장애 격리
- 특정 서비스에 오류가 발생하여도 다른 서비스에 무영향
